# SPENT — 假装下单机 / Fake Checkout for Impulse Cravings

> 一个让你完整体验"下单快感"、却一分钱不花的网站。
> 你 *spent* 的不是钱，是那股冲动本身。

---

## 0. 一句话定位

**SPENT 是一台"假装下单机"。** 当你冲动想买点什么（外卖、网购、奶茶、那双鞋……），
来这里走完一整套以假乱真的下单流程——挑选、加购、领券、付款、"下单成功"——
身体记忆被完整满足，冲动被"花掉"了，但钱包毫发无伤。

---

## 1. 名字：SPENT

一个词同时说清"花了"和"没花"：

- **spend 的过去式**——但你 *没有* spend 任何钱。
- **"I'm spent"**＝"我被掏空了 / 那股劲儿过去了"——你的**冲动**被 spent 了。

### 字母拆解（每个字母一个词，连起来就是产品的 point）

主选方案（收据风格，最后一行永远是 `Total: ¥0`，呼应"假装下单"）：

> **S**imulate the **P**urchase, **E**mpty the **N**eed, **T**otal: ¥0
> *模拟那次购买，清空那份渴望，总计：¥0*

- **S** — Simulate（模拟）
- **P** — Purchase（购买）
- **E** — Empty（清空 / 掏空）
- **N** — Need（渴望 / 需求）
- **T** — Total ¥0（总计零元）

备选拆解（更直白）：

- **S**imulate · **P**urchase · **E**xhaust · **N**eed · spend **N**o**T**hing
- **S**atisfy the urge, **P**ay **N**othing, **E**nd up **T**otally fine

> 待定：主选 vs 备选，最终文案要再打磨。`Total: ¥0` 这个收尾很贴产品（每张收据最后一行都是总价，而我们的永远是 0）。

## 2. 核心洞察（为什么这玩意儿真的有用）

你馋的不是食物 / 不是那件东西，**你馋的是"下单"这个动作本身。**

- 多巴胺主要在**预期 / 追逐阶段**释放，而不是**实际获得**阶段。
- 所以当你选好、加购、准备付款时，奖励回路其实**已经被喂饱了**——
  这就是"在点的那一瞬间就不饿了"。
- SPENT 做的，就是把这套"预期快感"完整复刻出来，然后**拦在真正花钱 / 吃下去之前**。
  等于给大脑一份**安慰剂订单**。

它不让你硬忍，而是**满足你**——只是用零成本的方式。这比"自律打卡 App"高明的地方。

---

## 3. 基调（重要：比例不能错）

- **主线 = 爽 / 沉浸 / 解压。** 前面的下单流程要做得足够真、足够爽。
- **彩蛋 = 治愈。** 结尾那下"会心一笑 + 松口气"是副产品，不是主体。
- **红线：不说教。** 结尾绝不能变成"你看你又冲动了吧"的爹味反转。要让人笑着松一口气。

## 4. 场景：先聚焦"食物 + 衣服（+ 美妆）"

> 决策（2026-06-24）：泛冲动太广，先缩范围。选食物和衣服——
> 本人和表妹"忍不住"的两样东西。美妆大概率顺手加上。

**为什么是这两样：它们正好覆盖两种完全不同的"下单手感"**，
一次性验证产品的两个核心形态：

- 🍜 **食物 = 外卖流**（美团 / 饿了么那套）
  选规格、加料、起送价、配送费、"预计 30 分钟送达"、骑手动画。
- 👗 **衣服 = 电商流**（淘宝 / 小红书那套）
  选色选码、"仅剩 3 件"、加购物车、店铺优惠券、付款。

这两条链的爽点和节奏不同，都要认真做真。

**💄 美妆 ≈ 免费加。** 口红 / 粉底的下单手感和买衣服几乎一样，都是电商流。
它**不是第三条新链，而是复用衣服那条链、换一套商品模板**而已。

➡️ **结论：MVP 只做两条链（外卖流 + 电商流），电商流下面挂"衣服 + 美妆"两类商品。**
工作量 = 2 条链，品类 = 3 个。

**实现方式：** 用户选品类 / 输入"想买啥"，系统用关键词 + 模板**拼出**以假乱真的商品页。
不需要真实商品数据库。

### 发布渠道 & 语言（决策 2026-06-24）

**大概率公开发小红书 + X**。后果：版权问题前置；转折页变成增长引擎（见 §5 第 4 段）。

- **语言：中英双语。** 一套网站支持中/英切换 → 小红书发中文版、X 发英文版。
  （`SPENT · Total: $0` 这种收据梗在英文语境里是天然 meme，X 上会更炸。）
- **商品来源：混合模型。**
  1. **现成示例商品**（低摩擦，进来立刻上手 / 爽）——用**自己画的 / 风格化**的商品图，
     **不用品牌官图**，规避版权；可秒刷秒点。
  2. **"贴你自己馋的那件"**（链接 / 截图，用户自带内容）——最准、最个人化，责任也在用户侧。
  - 上线版加一句"本站为玩笑 / 非真实商店"免责声明。

## 4.5 第一条赛道：外卖（决策 2026-06-24）

**先只做外卖这一条赛道**——不铺两条。理由：

- 小红书奖励垂直，"假装点外卖的网站"好记好传；
- 工作量砍半，省下的力气全砸在转折分享卡上；
- 核心引擎（假装下单的多巴胺）是通用的，一条赛道就能验证产品成不成立；
- 第二条赛道（电商/衣服美妆）复用 90% 代码，火了再补，决定可逆。
- 外卖是本人最原始最真实的灵感，故事钩子最锋利："假装点外卖，治好我的馋"。

### ⭐ 护城河：必须复刻"国内满减 / 会员 / 凑单"机制

> 没人做过。国外没这套；国内做"防剁手"的人只想让你别买，没人想到去**复刻满减的爽**。
> 而满减 / 凑单 / 会员红包，才是国内外卖多巴胺的**真正高潮**——
> 看着价格层层下掉、为凑满减疯狂加购的那个过程，比商品本身更上瘾。
> **把这个做真，比把商品做真更重要。**

要复刻的机制清单（越真越爽）：

- **会员红包 / 神券**：会员每月领的大额券、专享价标签。
- **店铺满减阶梯**：满 20 减 5 / 满 40 减 12 / 满 60 减 20。
- **凑单提示**（上瘾核心）："再买 ¥3.5 即享满减优惠" + 醒目的「去凑单」按钮。
- **起送价**："还差 ¥8 起送"。
- **新客立减 / 首单礼**。
- **红包膨胀**：点一下，红包"膨胀"动画 +金额。
- **配送费 + 满 X 免配送费**。
- **优惠叠加 → 价格层层下跌的动画**（满减 + 红包 + 会员，这是峰值快感）。
- **倒计时**："距优惠结束 09:59"，制造紧迫感。

### 💥 由满减引出的杀手锏转折

满减 / 凑单的本质 = **"你以为在省钱，其实被诱导多花了"**（为省 ¥5 去凑单，多花 ¥18）。
SPENT 先让你尽情凑单、爽够，转折页再一刀点醒：

> **"你刚才为了省 ¥5，差点多花 ¥18 来凑单。现在，¥0。"**

→ 满减机制本身就完美演绎了"冲动陷阱"。不是反对它，而是**先让人爽够，再让人看清**。
比单纯"省了多少钱"高一个段位。

## 5. 核心体验：完整的"爽链"

### 起点 · 带着冲动来
不是冷冰冰的输入框，而是一句很懂你的话：
> "这会儿，馋啥 / 想买啥？"

用户随便打——"螺蛳粉""那双鞋""一杯奶茶"。系统不评判，秒接住："好，咱们去下单。"

### 第 1 段 · 挑选（喂第一口多巴胺）
- 根据输入生成以假乱真的商品页：图、价格、"月销 10万+"、几条好评、"仅剩 3 件"。
- 快感来自"我在拥有它的路上了"。
- 细节越真越爽：滑动选规格、加料、选数量。

### 第 2 段 · 加购 + 凑单（**快感峰值**）
- 点"加入购物车"→ 右上角红色角标 `+1` 跳动（这个动画要做足）。
- 购物车金额往上跳。
- **杀手锏：国内满减 / 凑单 / 会员红包**（详见 §4.5 机制清单）。
  满减阶梯 + 凑单提示 + 红包膨胀，金额层层往下掉。
  → 省钱的多巴胺**比花钱更强**，这是整个产品最爽的 0.5 秒，也是护城河。

### 第 3 段 · 下单（**高潮**）
- 又大又满足的"提交订单"按钮。
- 点下去 → 微信 / 支付宝那种支付动画：转圈 →"叮"→ 绿色的勾打出来。
- 身体记忆被完整触发。这一刻，大脑里那笔订单已经"完成"了。

### 第 4 段 · 转折（**产品的灵魂**）
支付成功的勾，缓缓变成订单详情：

> **已送达：一次被你识破的冲动**
> 本单为你节省 **¥43 · 612 大卡 · 28 分钟等外卖的焦虑**
>
> *"其实你刚才不是饿，是想要那个'马上拥有'的感觉。现在有了。"*

爽感被满足、钱包毫发无伤、还顺手被治愈一下。

> ⚠️ **这一段是增长引擎，不只是灵魂。** 小红书 / X 的传播全靠用户截图这张转折页。
> 所以它要被设计成一张**为截图而生的分享卡**：
> - 构图像"收据 / 订单"，信息一眼能懂；
> - 角落带小小的 **SPENT** logo + 网址 → 别人转发 = 帮你导流；
> - 数字要戳人（省了 ¥X / X 大卡 / X 分钟）；
> - 中英双语各出一版（小红书用中文、X 用英文）。
> 这张卡 = 你的获客海报，值得单独打磨。

## 6. 留住用户的一层：战利品墙 🏆

- 每完成一次"假下单"，生成一张**收藏卡**（可复用卡片美学）。
- 卡片记下：那天馋的是什么、省了多少钱 / 多少卡 / 多少时间。
- 攒成一面墙 → "这个月我假装下单 23 次，虚拟省下 ¥800"。
- 把"忍住"变成**可炫耀的战绩**，而不是压抑的克制 → 给长期回访理由。

## 7. MVP 范围

**目标：一个周末跑通核心爽链。**

- ✅ 纯前端单页，无后端、无真实商品数据。
- ✅ **两条链：外卖流（食物）+ 电商流（衣服 / 美妆）。** 电商流是衣服、美妆共用一套。
- ✅ 流程：选品类 / 输入冲动 → 挑选 → 加购 → 优惠券 → 下单 → 转折。
- ✅ 商品页用模板 + 关键词拼出。
- ✅ 转折页 + 一句核心文案。
- 🔜 战利品墙（可用 localStorage 存，先不做账号系统）。
- 🔜 截图 / 分享卡片。

### 暂定技术栈
- 纯 HTML / CSS / JS 单页起步（最快出原型）；
- 若要做大可迁到 Vite + 组件化（参考 prefrontal 项目的迁移路径）；
- 部署：Cloudflare Pages / GitHub Pages（参考 tarot 项目）。

## 8. 风险 & 注意

- **转折说教化**：守住"爽是主线、治愈是彩蛋"的比例。
- **逼真度不够则不爽**：商品页 / 支付动画的细节决定成败，宁可少做功能也要把这条链做真。
- **新鲜感衰减**：第一次最爽，重复来怎么保持？→ 靠战利品墙的收集感 + 商品页 / 文案的随机性。
